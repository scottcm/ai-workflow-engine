# Contributing to AI Workflow Engine

Thank you for your interest in contributing.  
This repository is a portfolio project designed to demonstrate clean architecture, extensibility, and design patterns in an AI-assisted development workflow engine.

While external contributions are welcome, the project maintains a consistent architectural direction.  
Please follow the guidelines below.

## How to Contribute

### 1. Issues
Before opening an issue, please:
- Search existing issues to avoid duplicates.
- Use the appropriate **Issue Forms** (Bug Report, Feature Request, Architecture Question).

### 2. Pull Requests
When submitting a pull request:
- Create a feature branch (`feature/...`, `fix/...`, or `refactor/...`).
- Reference the related issue in your PR description.
- Ensure the code aligns with project architecture and patterns (Strategy, Factory, CoR, Builder, Command, Adapter, Template Method).
- Keep changes focused and scoped; avoid unrelated modifications.

### 3. Style & Quality
- Use Python 3.11+.
- Follow Pydantic conventions for models.
- Use type annotations throughout.
- Write clear, intention-revealing code.
- Add tests when appropriate.

### 4. Discussions
For architectural exploration, design decisions, or high-level questions, please use **GitHub Discussions** (Architecture or ADR categories).

Thank you for helping improve the project.
